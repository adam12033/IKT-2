A kialakított rendszerben van három Debian Linux operációs rendszerû számítógép:

Szerver
Client1
Client2

Amelyekbõl Szerver olyan, hogy 2 db NIC, egyik NAT, a másik Belsõ csatoló, hogy így lehessen egy VirtualBox-on belüli LAN átjárója (Client1,2 rajta keresztül fér internethez)
Ez a Szerver ráadásul DHCP funkciót is ellát így a Belsõ csatolóján: 192.168.100.254/25 <=> 255.255.255.128 keresztül tud IPv4 címeket és egyéb járulékos adatokat (subnet mask, default gateway, DNS, ...) a Client1 és Client2 részére a 192.168.100.129-tõl 192.168.100.192-ig terjedõ természetesen 255.255.255.128 alhálózati maszkokkal
A Szerver az elõzõeken túl még Active Directory, centralizált, adatbázis-alapú hálózatadminisztrációs szolgáltatást is biztosít a Server Message Block protokoll felett.
Mindezek alapján mivel az Active Directory által meghatározott tartományi környezetbe felvehetõk a kliensek, ideális esetben azok nemcsak IPv4 címük, hanem DNS backend (ami a példában a SAMBA_INTERNAL volt) következtében név alapján is lehetõvé teszi azok elérését.

Mivel probléma mutatkozott a Linux kliensek lényegében Microsoft-os tartományi környezetbe történõ beléptetése során ezt lényegesen egyszerûbben tanulmányozhatjuk Windows Desktop oprációs rendszerû virtuális gépet alkalmazva
(Windows-os gép tartományba léptetése a Számítógép nevének módosítására szolgáló párbeszédpanel Tartomány rovata kitöltésével lehetséges)

A tartományba léptetés fontos feltétele, hogy a tartományra való szöveges hivatkozás a háttérben IPv4 címre legyen átalakítható (balintdezso.local <=> 192.168.100.254) amit ráadásul a tartományi környezet Administrator userével kell hitelesítenünk.
(Siker esetén az adott pl. Windows-os gép máris bekerül a tartomány Computer-ei közé, amit a PDC ~ primary domain controller/elsõdleges tartományvezérlõn listázhatunk is:

samba-tool computer list

CLIENT1$
SZERVER$
CLIENT2$
CLIENT3$

Ekkor azonban nem történik meg automatikusan a DNS-backend-ben a gépnév A DNS rekordként történõ felvétele, ami természetesen a gépnévhez hozzárendelné az épp aktuálisan birtokolt IPv4 címet, hogy innentõl kezdve név alapján is legyen elérhetõ.

A PDC-n manuálisan megtehetjük ezt:

samba-tool dns add szerver balintdezso.local client3 a 192.168.100.133 -U Administrator

(mivel a NAT-olt kártyája a PDC-nek a sulis hálózat DNS szervereit tartalmazza az /etc/resolv.conf-ban így kénytelenek leszünk ezt kiegészíeni: nameserver 192.168.100.254 és nameserver 10.0.2.15 bejegyzésekkel, hogy maga a PDC, mint DNS szerver kaphasson szerepet a ping client3.balintdezso.local végrehajtásakor)
